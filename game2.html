Zombie Tower Defence

Published Project

After my first mini game, I wanted to create a more complex game. I have always loved playing tower defense games, but every time I played them, I wondered what it would be like to play in an FPS mode and also be the defender. This project took around 5 months to complete from scratch. During this project, I learned a lot. I was inspired by several game developer YouTubers such as Brackeys, Codemonkey, and Jason Weinmann. I added many features and modified the code to meet my specific needs. The features I designed, coded, and tested included the dropping system, reload animation, Unity ads implementation, and UI system. I also learned a lot about how to communicate between scripts, and I created the level design and menu design. My wife was my UI designer, she made all the visuals, so we collaborated on this aspect of the project. If I were to do this project again, I would change the lighting and implement object pooling. The lighting should have been done by baking to be more performant, especially on mobile platforms, and I would also switch to URP for the same reason. After a year and a half, I received around 1300 downloads, and the game did not perform as I thought it will. Platform: Exclusively available for Android devices

Genre: Hybrid tower defense and first-person shooter

Target audience: asual players who enjoy strategic and action-packed gameplay

Engine and language: Developed using Unity engine and C# programming language

Key Features and Development Process:

Innovative gameplay mechanics: Combined elements of tower defense and first-person shooter genres, enabling players to strategically place turrets and actively defend as one of the on-screen characters in FPS mode

Efficient pathfinding system: Overcame performance challenges by adopting a waypoint system for enemy navigation, as Unity's NavMesh system was too resource-intensive for mobile devices.

Comprehensive turret system: Developed a robust system for managing different turret types, including spawning, ammo tracking, and seamless interaction with the player.

3D game with free and modified assets: Utilized free assets from the Unity Asset Store and tailored them to fit the game's unique aesthetic and design requirements.

Character and enemy animations: Leveraged Adobe Mixamo to create dynamic animations for player characters and enemies.

Immersive audio design: Overcame audio challenges by carefully mixing various sound elements, such as enemy walking sounds, weapon firing sounds, background music, ambient sounds, and UI sounds.

Collaborative UI design: Worked with my wife, who crafted the art style in Photoshop, while I implemented the UI elements within Unity.

Optimization techniques: Baked lighting, texture compression, and an optimized pathfinding system to ensure smooth gameplay on mobile devices.

Programming tools: Utilized programming patterns such as singleton pattern.

Project Timeline and Post-launch Support:

Six-month development: Successfully completed the project from conception to publication within a six-month timeframe.

Bug fixes and updates: Addressed post-launch issues, including animation glitches, UI malfunctions, and introduced an additional map to enhance gameplay.

Areas for Future Improvement:

Art design: Received mixed reviews, with players praising the game's mechanics but critiquing the visual aspects. As a programmer-focused developer, I am continuously working to improve my game's visual appeal.
